# 🧑‍💻 My Scrollytelling Portfolio

> スクロールで読み進める物語\
> 静的なウェブサイトを超えて、ユーザーに視覚的な楽しさを与えるスクロールテリング手法を取り入れたインタラクティブな自己紹介ページです。

🔗 **[公開されたウェブサイトを見る](https://github.com/interlude113/duzzonku/tree/main/Introduce_syu)**

---

## 🛠 Tech Stack

- **Core:** HTML5, CSS3, JavaScript (Vanilla)
- **Library:** AOS (Animate On Scroll)

---

## ✨ 主な機能

1. **スクロールアニメーション**
   - ユーザーのスクロール位置に反応して、要素がスムーズに登場します。
   - `data-aos-delay`を活用して情報が順番に視界に入るようにし、視線の分散を防いで可読性を高めました。
2. **モダンCSSレイアウト**
   - 従来の`float`方式から脱却し、**Flexbox**と**CSS Grid**を活用して、直感的で保守しやすいレイアウトを構成しました。
3. **レスポンシブウェブ (Responsive Web)**
   - Media Query(`@media`)を適用し、PC、タブレット、モバイルなど様々なデバイス環境でもUIが崩れないよう最適化しました。

---

## 💡 工夫した点と解決のプロセス

**1. "要素が一度に現れて散漫に見える問題"**

- **問題:** スクロールを下ろした時、画面内のテキストと画像が同時に飛び出してきて視覚的な疲労感を与える。
- **解決:** AOSライブラリの`delay`属性を200ms単位でずらして付与。
  - (例: `画像登場` -> `0.2秒後にタイトル登場` -> `0.4秒後に説明登場`)
  - これにより、ユーザーが自然に情報の流れを追えるようUXを改善。

**2. "外部ライブラリ(AOS)への依存度に対する考察"**

- ライブラリを使えば実装は早いが、内部の原理を知らないとエラー対応が難しいことに気づいた。
- 今後はVanilla JSの`Intersection Observer API`を学習し、重い外部ライブラリを使わずにスクロールイベントを自ら最適化して実装してみる予定。
